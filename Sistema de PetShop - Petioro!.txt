Nosso trabalho possui as pastas css, html e img, o arquivo PDF Petioro e o README do GitHub.

- O documento Petioro.pdf é o "rascunho" que fizemos do sistema para começar a desenvolver o mock up!

- O arquivo README.md é automaticamente gerado pelo Github, ferramenta que usamos para versionar nosso sistema.

- Na pasta CSS temos o arquivo styleguide.css com todo o estilo utilizado no sistema inteiro.

- Na pasta img temos todas as imagens utilizadas na produção do mock up.

- Na pasta html temos os seguintes arquivos fonte:
	- adm-adms: página do administrador reponsável por mostrar os administradores cadastrados.
	- adm-calendar: página do administrador que contém o calendáro.
	- adm-clients: página do admistrador responsável por mostrar os clientes cadastrados.
	- adm-earnings: página do administrador que exibe os ganhos da empresa em determinado mês.
	- adm-edit-adm: página de edição dos administradores cadastrados.
	- adm-edit-client: página de edição dos clientes cadastrados.
	- adm-edit-product: página de edição dos produtos cadastrados.
	- adm-edit-service: página de edição dos serviços cadastrados.
	- adm-login: tela de login.
	- adm-new-adm: página de inserção de um novo administrador.
	- adm-new-client: página de inserção de um novo cliente.
	- adm-new-product: página de inserção de um novo produto.
	- adm-new-service: página de inserção de um novo serviço.
	- adm-products: página que exibe a relação de produtos disponíveis na loja.
	- adm-sales: página com a interface administrador para realizar uma venda pessoalmente.
	- adm-sales-close: página para finalizar a venda e escolher a forma de pagamento.
	- client-calendar: página para o cliente poder visualizar os eventos marcados.
	- client-calendar-new-event: página para cadastrar um novo evento. Essa página na verdade é da interface do ADMINISTRADOR, mas só entendemos depois que clientes não agendam horários, não mudamos o nome para não quebrar os links! Porém a página está configurada e sendo mostrada com a interface de administrador!
	-client-cart: página que mostra o carrinho do cliente.
	- client-edit-pet: página para que o cliente possa editar os dados do seu pet.
	- client-new-pet: página para que o cliente possa cadastrar um novo pet.
	- client-pets: página que exibe os pets cadastrados do cliente.
	- client-products: página para que o cliente realize compras online.
	- style-guide: página amostra dos estilos utilizados no sistema!

	**Fica mais bonito no Safari, IE, Chrome e Opera devido ao estilo do checkbox!
